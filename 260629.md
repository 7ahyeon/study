*vi*
- i : INSERT 모드 : 문자 편집
- esc : COMMAND 모드 : 복사/붙여넣기/파일저장/종료 등
- :q + enter : 종료
- wq + enter : 저장 종료
- wq! + enter : 저장 강제 종료
- dd : 줄 삭제
- yy : 복사
- p : 붙여넣기
- u : 되돌리기
- :set number : 라인 출력
- /검색어 n : 텍스트 찾기
- :%s/원본/대체/g : 텍스트 대체

- cat : 파일 내용 확인/출
- grep : 출력 내용 검색
- cat 파일명 | grep 검색어 : 파일 내용 검색
- grep -B n 검색어 : 파일 내용 검색 + 이전 n개 라인 같이 출력
- grep -A n 검색어 : 파일 내용 검색 + 이후 n개 라인 같이 출력
- grep -i 검색어 : 대소문자 없이 검색
- 이전 명령어 > 파일명 : 이전 명령어 출력을 파일로 저장(기존 파일 있을 경우 덮어쓰기)
- 이전 명령어 >> 파일명 : 이전 명령어 출력을 파일로 저장(기존 파일 있을 경우 내용 추가)
- less : 용량 큰 파일 읽기(읽기 전용/편집x)

- tar : 아카이브 (여러 디렉터리/파일 하나의 파일로 묶음/압축X)
- tar -cvf {파일명.tar}{디렉터리/파일} : 파일/디렉터리 아카이브 생성
- tar -xvf {파일명.tar} : 파일/디렉터리 해제
- tar.gz : 아카이브 압축
- tar -czvf {파일명.tar.gz}{디렉터리/파일} : 파일/디렉터리 아카이브 압축 생성
- tar -xzvf {파일명.tar.gz}{디렉터리/파일} : 파일/디렉터리 아카이브 압축 해제

*Git*
- git revert : 변경 사항 취소하는 새로운 커밋 생성
- git reset : 변경 사항 삭제함(push 충돌 가능성O)
- git restore : add하지 않은 파일 변경사항 취소
- git restore --staged : 스테이징된(add된) 파일 취소
- git restore --source=헤드/커밋해시 파일명 : 파일 커밋 상태로 되돌리기

- git merge : 병합 (기존 커밋 유지)
- git rebase : 브랜치 커밋을 다른 브랜치 끝(최신 커밋)으로 이식함 (신규 커밋으로 재작성됨) (push 충돌 가능성O)

- git merge/rebase --abort : 되돌리기

- git push : 로컬에서 원격으로 커밋
- git push --delete 원격명 태그명 : 태그 삭제
- git push 원격명 --tags : 로컬 모든 태그 원격 올리
- git pull : 원격에서 로컬로 당겨오기(merge/rebase 포함)
- git fetch : 원격에서 로컬로 가져오기

- git push/pull --force : 강제

- git branch --all : 브랜치 확인
- git switch -t : 원격 브랜치 추적하는 로컬 브랜치 생성 및 즉시 이동
- git push 원격이름 --delete 원격브랜치명 : 원격 브랜치 삭제

- git add : 스테이징 영역 추가(커밋 전 단계)

- git checkout : 브랜치 이동
- git checkout -b : 새 브랜치 생성 및 이동
- git checkout 커밋해시ID : 커밋 시점 상태 확인
- git checkout -- 파일명 : 파일 마지막 커밋상태 복원
- git checkout. : 작업데릭터리 모든 수정사항 취소

- GIT HEAD : 현재 속한 브랜치 가장 최신 커밋
- git checkout HEAD^^^/HEAD~5 : ^/~개수만큼 이전으로 이동
- git checkout - : 이동 한 단계 되돌리기

- j : 내리기
- k : 올리기

- 용어 권장
- feat : 새로운 기능 추가
- perf : 성능 개선
- chore : 빌드 과정 또는 보조 기능(문서생성) 수정

- git add -p hunk단위 스테이징
- git stash : 임시 저장
- git stash pop : 임시 저장 적용(저장내용삭제)
- git commit --amend : 마지막 커밋 수정
- git rebase -i : 대상 바로 이전 커밋
- p pick : 커밋 그대로 두기
- r reword : 커밋 메시지 변경
- e edit : 수정 위해 정지
- d drop : 커밋 삭제
- s squash : 이전 커밋 합치기(메시지 새로 작성)
- f fixup : 이전 커밋 합치기(메시지 이전 커밋 건으로 통일)

- git clean : 삭제
- -n : 삭제 파일 조회
- -i : 인터랙티브 모드 시작
- -d : 폴더 포함
- -f : 강제 삭제
- -x : .gitignore 파일 포함 삭제
- git clean -fd/df : 파일 폴더 모두 삭제
- git clean -fdx : 마지막 커밋 상태로 초기화(빌드 파일 포함)

- git reflog : reset/rebase한 내역 포함 전체 변경 이력 기록 확인(복구 가능)
- git log : 현재 커밋 트리 히스토리

- git tag : 특정 시점 키워드 저장/버전정보 추가
- 시맨틱버저닝(SemVer : v1.0.0) / 캘린더버저닝(CalVer : v2026.06.29)
- git show 태그명 : 태그내용 확인
- git tag -d : 태그 삭제
- git tag -l : 태그 전체 조회
- git tag -l ''v1.*' : 태그 패턴 필터링 조회

- git cherry-pick : 다른 브랜치의 원하는 커밋만 가져오기
- git rebase --onto 도착(메인) 출발(과일) 이동(딸기) 브랜치 : 과일 브랜치에서 파생된 딸기 브랜치를 메인 브랜치로 옮겨붙이기

- git reset --hard : 현재 작업 내용 및 커밋 이력을 특정 시점으로 강제로 되돌림(덮어쓰기)
- git merge --squash : 다른 커밋들을 하나로 묶어서 가져오기(merge X : 접붙일 브랜치 복사해서 한 마디로 모아서 붙임staged 상태)


*브랜치 전략*
- GIT FLOW : main develop feature release hotfix : 엄격한 관리 및 안정성
- GITHUB FLOW : main중심(main topic(feature)) : 단순함 빠른 배포
- TBD (Trunk-based Development) : 모든 개발자가 main(Trunk)에 매일 자주 병합 : 지속적 통합 및 속도

- git diff : 가장 최근 커밋과 현재작업(워킹 디렉토리) 변경 사항 확인
- git diff --stat : 변경 내용X 어떤 파일 몇 줄 바뀐지 통계
- git diff w : 공백 차이 무시
- --color-words : 단어 단위 바뀐부분 색강조
- --name-only 변경 파일명만 확인
- --staged : 스테이지된 변경사항 확인
- git diff  커밋/브랜치1 커밋/브랜치2 : 커밋/브랜치 간 차이 확인

- git blame 파일명 : 파일 부분별 작성자 확인
- git blame -e 파일명 : 파일 부분별 작성자이메일 확인
- git blame -L 시작줄 끝줄/+줄수 파일명 : 특정 부분 작성자 확인

- git bisect : 이진 탐색 알고리즘 사용 버그/문제 발생 특정 커밋 찾아주는 git 디버깅 도
- git bisect start 시작 - git bisect bad 오류발생지점 표시- git checkout 커밋해시 이동 - git bisect good 오류없음 표시
- 원인 찾을때까지 git bisect bad/good 반복
- git bisect reset 이진 탐색 종료

- Git Hooks 커밋/푸시/병합 등 특정 이벤트 발생시 사용자 지정 스크립트 자동 실행
- .git/hooks 위치
- git worktree : 여러 브랜치 동시 작업

- Pull request : 변경사항 merge/commit 전 팀원 리뷰 요청함
- 반려 ; close pull request
- 승인 ; merge pull request

- fork : 다른 사용자 원격 저장소 통째로 복사 : 오픈 소스 협업
- 코드 수정 후 pull request : 코드 기여
